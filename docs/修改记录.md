## 问题描述

用户在使用月结账单功能时，点击"生成账单"并选择了特定商品类别，但生成的账单仍包含了其他类别的商品。

## 问题分析

### 问题定位
通过服务器日志分析发现：
```
===== 生成月结账单 =====
客户ID: 6
账单月份: 2026-03
类别IDs: null  // 问题：前端传递的 categoryIds 为 null
```

### 根本原因
**Axios 数组参数序列化格式不兼容 Spring Boot**

- **Axios 默认行为**：将数组参数序列化为 `categoryIds[]=1&categoryIds[]=2` 格式
- **Spring Boot 期望**：接收 `categoryIds=1&categoryIds=2` 格式（重复参数名）
- **后端接收**：使用 `@RequestParam(required = false) java.util.List<Long> categoryIds`

由于格式不匹配，Spring Boot 无法正确解析数组参数，导致接收到的 categoryIds 为 null。

## 解决方案

### 修改文件：`frontend/src/utils/request.js`

**修改内容**：在 Axios 实例配置中添加数组参数序列化配置

```javascript
const request = axios.create({
  baseURL: '/api',
  timeout: 10000,
  paramsSerializer: {
    // 配置数组参数序列化格式，使用 repeat 格式（a=1&a=2）以兼容 Spring Boot
    indexes: null  // 这会生成 a=1&a=2 而不是 a[0]=1&a[1]=2
  }
})
```

**工作原理**：
- `indexes: null` 配置使 Axios 生成重复参数名的格式
- Spring Boot 能正确解析为 List 类型参数

### 附加修复

在测试过程中发现 `echarts` 依赖缺失导致构建失败，已通过 `npm install echarts` 修复。

- 前端：Docker 容器（端口 80）

```
用户浏览器 (HTTPS)

    ↓
前端容器 (80端口) ← 静态文件
    ↓
后端容器 (8080端口) ← API请求
    ↓
MySQL容器 (3306端口)
```

**错误信息**：
```

```

- **原因**：Nginx 运行在宿主机，无法解析 Docker 容器名称

2. Nginx 站点配置 `/etc/nginx/conf.d/oil.conf` 中有 HTTP 重定向 server 块监听 80 端口

**解决方案**：
1. 注释掉 `/etc/nginx/nginx.conf` 中的默认 server 块（第 39-53 行）
2. 注释掉 `/etc/nginx/conf.d/oil.conf` 中的 HTTP 重定向 server 块（第 5-12 行）

## 新增功能：成本和利润管理

### 需求背景
用户需要记录商品成本并自动计算订单利润，但希望成本数据仅管理员可见，同时保持系统无需登录的便捷性。

### 功能概述
- 商品可选填成本价
- 系统自动计算订单利润
- 新增利润统计分析
- 管理员权限控制（成本/利润数据仅管理员可见）
- 保持普通模式无需登录

### 1. 数据库变更

#### 修改内容：
**新增字段**：
```sql
-- product 表
ALTER TABLE product ADD COLUMN cost DECIMAL(10,2) DEFAULT NULL COMMENT '成本价（可选）';

-- order_item 表
ALTER TABLE order_item ADD COLUMN cost DECIMAL(10,2) DEFAULT NULL COMMENT '成本价快照（可选）';
ALTER TABLE order_item ADD COLUMN profit DECIMAL(10,2) DEFAULT NULL COMMENT '单品利润';
```

**设计说明**：
- 成本字段允许 NULL，支持不填写成本
- order_item 记录成本快照，避免后续修改商品成本影响已有订单
