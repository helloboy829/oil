  ```java
  private List<CategoryWithCountDTO> buildTree(List<CategoryWithCountDTO> allCategories, Long parentId) {
      return allCategories.stream()
          .filter(category -> {
              if (parentId == null) {
                  return category.getParentId() == null;
              }
              return parentId.equals(category.getParentId());
          })
          .peek(category -> {
              List<CategoryWithCountDTO> children = buildTree(allCategories, category.getId());
              category.setChildren(children.isEmpty() ? null : children);
          })
          .collect(Collectors.toList());
  }
  ```

### 10. 数据统计 - 修复库存统计类型转换错误

#### 问题描述
库存统计模块上线后出现运行时错误：
```
class java.math.BigDecimal cannot be cast to class java.lang.Long
```

#### 问题分析
- MySQL 的 `SUM()` 和 `COUNT()` 聚合函数返回 `BigDecimal` 类型
- 代码中尝试直接将 Map 中的值转换为 `Long` 类型
- Java 不允许 `BigDecimal` 直接转换为 `Long`，导致 `ClassCastException`

#### 修复方案

**文件：`backend/src/main/java/com/oil/system/controller/StatisticsController.java`**

**修改位置**：第 272-274 行

**修改前**：
```java
Long stock = (Long) map.get("stock");
Long productCount = (Long) map.get("productCount");
```

**修改后**：
```java
// MySQL 的 SUM/COUNT 返回 BigDecimal，需要转换
Number stock = (Number) map.get("stock");
Number productCount = (Number) map.get("productCount");
```

**关键改动**：
- 使用 `Number` 类型（`Long` 和 `BigDecimal` 的公共父类）接收 SQL 查询结果
- 通过 `intValue()` 方法安全转换为 `Integer`
- 适配 MySQL 聚合函数的返回类型

#### 技术说明

**为什么 MySQL 返回 BigDecimal？**
- MyBatis 将 MySQL 的 `DECIMAL` 类型映射为 Java 的 `BigDecimal`
- `SUM(stock)` 和 `COUNT(*)` 在 MySQL 中返回数值类型，默认映射为 `BigDecimal`
- 即使 `stock` 列是 `INT` 类型，聚合后仍返回 `BigDecimal`

**解决方案选择**：
1. ✅ 使用 `Number` 类型接收（推荐）- 兼容多种数值类型
2. ❌ 修改 SQL 使用 `CAST()`  - 增加 SQL 复杂度
3. ❌ 修改 MyBatis 配置 - 影响全局行为

#### 测试验证
- ✅ 本地测试通过
- ✅ 服务器部署成功
- ✅ 库存统计页面正常显示

### 7. MySQL 字符编码问题修复

#### 问题描述
批量导入蓄电池数据后，中文显示为乱码（无论在终端还是前端）。

#### 问题分析
- MySQL 客户端默认使用 `latin1` 字符集连接数据库
- 数据库、表、列都配置为 `utf8mb4`，但客户端连接字符集不匹配
- 导致中文数据写入时被错误编码

**验证命令**：
```bash
docker exec -it oil-mysql mysql -uroot -p123456 \
  -e "SHOW VARIABLES LIKE 'character%';"
```

**结果显示**：
```
character_set_client    | latin1  # ❌ 客户端使用 latin1
character_set_connection| latin1  # ❌ 连接使用 latin1
character_set_database  | utf8mb4 # ✅ 数据库使用 utf8mb4
character_set_results   | latin1  # ❌ 结果集使用 latin1
```

#### 解决方案

**修复步骤**：
1. 删除乱码数据：
```bash
docker exec -it oil-mysql mysql -uroot -p123456 \
  --default-character-set=utf8mb4 \
  -D oil_system \
  -e "DELETE FROM product WHERE code LIKE 'BAT%';"
```

2. 重新导入（指定正确字符集）：
```bash
docker exec -i oil-mysql mysql -uroot -p123456 \
  --default-character-set=utf8mb4 \
  -D oil_system < scripts/batteries-import.sql
```

3. 验证数据：
```bash
docker exec -it oil-mysql mysql -uroot -p123456 \
  --default-character-set=utf8mb4 \
  -D oil_system \
  -e "SELECT id, name, price FROM product WHERE code LIKE 'BAT%' LIMIT 3;"
```

**验证结果**：
```
id  | name                  | price
----|----------------------|-------
36  | 能洋36V反小头          | 190.00
37  | 能洋45L反大头          | 200.00
38  | 银标瓦尔塔65D23L反     | 370.00
```

## 问题描述

用户在使用月结账单功能时，点击"生成账单"并选择了特定商品类别，但生成的账单仍包含了其他类别的商品。

## 问题分析

### 问题定位
通过服务器日志分析发现：
```
===== 生成月结账单 =====
客户ID: 6
账单月份: 2026-03
类别IDs: null  // 问题：前端传递的 categoryIds 为 null
```

### 根本原因
**Axios 数组参数序列化格式不兼容 Spring Boot**

- **Axios 默认行为**：将数组参数序列化为 `categoryIds[]=1&categoryIds[]=2` 格式
- **Spring Boot 期望**：接收 `categoryIds=1&categoryIds=2` 格式（重复参数名）
- **后端接收**：使用 `@RequestParam(required = false) java.util.List<Long> categoryIds`

由于格式不匹配，Spring Boot 无法正确解析数组参数，导致接收到的 categoryIds 为 null。

## 解决方案

### 修改文件：`frontend/src/utils/request.js`

**修改内容**：在 Axios 实例配置中添加数组参数序列化配置

```javascript
const request = axios.create({
  baseURL: '/api',
  timeout: 10000,
  paramsSerializer: {
    // 配置数组参数序列化格式，使用 repeat 格式（a=1&a=2）以兼容 Spring Boot
    indexes: null  // 这会生成 a=1&a=2 而不是 a[0]=1&a[1]=2
  }
})
```

**工作原理**：
- `indexes: null` 配置使 Axios 生成重复参数名的格式
- Spring Boot 能正确解析为 List 类型参数

### 附加修复

在测试过程中发现 `echarts` 依赖缺失导致构建失败，已通过 `npm install echarts` 修复。

### 需求背景
用户需要记录商品成本并自动计算订单利润，但希望成本数据仅管理员可见，同时保持系统无需登录的便捷性。

### 功能概述
- 商品可选填成本价
- 系统自动计算订单利润
- 新增利润统计分析

### 1. 数据库变更

#### 修改内容：
**新增字段**：
```sql
-- product 表
ALTER TABLE product ADD COLUMN cost DECIMAL(10,2) DEFAULT NULL COMMENT '成本价（可选）';

-- order_item 表
ALTER TABLE order_item ADD COLUMN cost DECIMAL(10,2) DEFAULT NULL COMMENT '成本价快照（可选）';
ALTER TABLE order_item ADD COLUMN profit DECIMAL(10,2) DEFAULT NULL COMMENT '单品利润';
```

**设计说明**：
- 成本字段允许 NULL，支持不填写成本
